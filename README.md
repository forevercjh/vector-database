# 关系型数据库 vs 向量数据库
| 特性             | 关系型数据库                                 | 向量数据库                                                     |
|------------------|----------------------------------------------|------------------------------------------------------------------|
| 数据类型         | 数值、字符串、时间等传统数据类型              | 向量数据，不存储原始数据                                       |
| 数据规模         | 小，1亿数据量为规模很大                       | 大，千亿数据是底线                                             |
| 数据组织方式     | 基于表格，按照行和列组织                      | 基于向量，按照向量维度组织                                     |
| 查找方式         | 精确查找：点查/范围查                         | 近似查找：对算力要求较高                                       |
| 低时延，高并发   | 否                                           | 是                                                             |
| 支撑上层应用     | 较弱                                         | 对外提供统一的 API，更适合大规模 AI 引用程序的部署和使用       |
| 应用场景         | 容错率低，需提供更为精准的搜索结果            |                                                                |
